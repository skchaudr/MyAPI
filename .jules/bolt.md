## 2025-05-18 - Avoid repeated re.compile and tight python loops in text processing
**Learning:** `re.compile` placed inside a frequently called function acts as a measurable bottleneck due to Python calling it repeatedly, even though `re` caches internally. Furthermore, performing line-by-line whitespace normalization in a Python loop (`text.split('\n')` combined with per-line regexes and slicing) is severely slower than executing a single well-crafted global regex (`re.sub` using `re.MULTILINE` or lookbehinds) across the entire string at C-speed. Finally, generator expressions inside `sum(c.isalnum() for c in s)` are much slower than mapping builtins directly `sum(map(str.isalnum, s))` due to generator overhead in Python.
**Action:** Always extract and pre-compile regular expressions to module-level constants. When manipulating large blocks of text, try to frame the transformation as a single global `re.sub` operation rather than splitting strings and processing lines sequentially in Python. Use `map` with builtins instead of generator expressions for simple counting tasks.
