## 2025-05-18 - Retrieval Pipeline Bottlenecks
**Learning:** The retrieval pipeline (`context_refinery/retrieval.py`) had severe CPU bottlenecks: `KeywordSearcher.search` was recompiling regular expressions for every document in a tight loop, and `ResultReranker._reinforcement_score` computed tag overlaps using an O(N^2) comparison across all documents. Additionally, `ResultReranker._title_body_path_score` eagerly compiled regex substitutions (`re.sub`) for anchor checking even when fast string matching (`in`) would have sufficed.
**Action:** Precompile regular expressions before entering inner loops. Use a precomputed inverted index (`tag_to_docs`) for overlapping sets to drop complexity from O(N^2) to O(N). Always prefer fast-path exact string matching (`in`) before falling back to lazy-initialized regex substitutions.
