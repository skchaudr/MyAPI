## 2024-05-23 - Optimize ResultReranker Reinforcement Score
**Learning:** In the retrieval pipeline (`context_refinery/retrieval.py`), `_reinforcement_score` historically exhibited O(N²) complexity, becoming a severe bottleneck (taking ~2.6 seconds for 2000 documents) due to nested loops iterating over all documents' tags. This occurs because the pipeline performs reinforcement scoring over potentially over-fetched collections of results before filtering and final aggregation.
**Action:** When working on reinforcement/grouping computations over large document sets, implement precomputed inverted indices (e.g., `tag_to_docs`) mapping metadata keys to document sets instead of iterating document-by-document. This reduces set intersection lookups to O(N * K) or practically O(1) per document-tag match, dropping latency from seconds to milliseconds.
