## 2024-05-24 - Pre-compile Regexes and Leverage Global Replacement in Text Processing
**Learning:** String line-iteration via `split('\n')` inside normalization loops is significantly slower than executing multi-line regular expressions across whole blocks of text. Constructing list aggregates in python introduces heavy memory overhead compared to `re.sub` string processing. Also, generating generator expressions vs `map()` calls for simple built-ins like `str.isalnum` adds measurable overhead.
**Action:** Always pre-compile `re.compile()` logic into module-level constants for frequently accessed string normalization methods. Optimize large text blocks using multi-line substitutions `re.sub(..., flags=re.MULTILINE)` to circumvent creating and destroying numerous tiny intermediate strings in a Python `for` loop. For iterative checking, prefer mapping fast builtins (like `sum(map(str.isalnum, text))`).
