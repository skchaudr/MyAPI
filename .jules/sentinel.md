## 2024-05-20 - Error handling exposing internal details\n**Vulnerability:** HTTP 500 error responses were exposing internal exception strings directly to clients.\n**Learning:** The FastAPI exception handling was returning `str(e)` verbatim, which could inadvertently expose stack traces, file paths, or sensitive internal logic.\n**Prevention:** Replace raw exceptions in HTTP 500 responses with generic messages ('An internal server error occurred') and use `logger.error(..., exc_info=True)` to preserve the stack trace in server-side logs without leaking it to the client.
