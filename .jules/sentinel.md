## 2026-04-12 - Fix eval command injection in deploy script
**Vulnerability:** The deployment script `deploy_to_brain.sh` contained an unsafe `eval` statement designed to expand the tilde (`~`) character when evaluating user input for a directory path: `eval LOCAL_DIR="$LOCAL_DIR"`. This introduced a critical command injection vulnerability. A malicious actor could provide input like `"; ls -al; echo "` to execute arbitrary commands with the privileges of the script user.
**Learning:** Shell scripts processing user input should avoid the `eval` builtin wherever possible as it evaluates arbitrary code. While `eval` is often tempting for tasks like tilde expansion, safer alternatives exist in bash.
**Prevention:** Rather than utilizing `eval`, use safe bash parameter expansion constructs. In this case, `LOCAL_DIR="${LOCAL_DIR/#\~/$HOME}"` performs a simple pattern substitution, replacing a leading tilde with the user's home directory path without executing the input as a command.

## 2026-04-13 - Prevent Path Traversal in API import endpoints
**Vulnerability:** The API endpoints `/codex` and `/claude-code` allowed arbitrary path imports due to a lack of input validation on the `root` parameter in the request body. While `os.path.expanduser` was used internally by the adapters, it did not enforce boundaries. This opened the risk of an attacker forcing the app to scan/import unintended directories from the host server. Furthermore, the generic catch-all exception blocks leaked raw stringified internal error details in 500 response bodies instead of failing securely.
**Learning:** Checking path boundaries string-wise (like using `.startswith()`) is unsafe because `/safe-dir-spoof` starts with `/safe-dir`. The secure mechanism involves `os.path.commonpath([base_dir, target_path]) == base_dir` after resolving and expanding both paths dynamically.
**Prevention:** For APIs importing files by path parameter, explicitly resolve `os.path.abspath(os.path.expanduser(...))` for both the permitted base directory and user input, then use `os.path.commonpath` to ensure the target falls within bounds. Finally, ensure blanket exceptions return safe, generic error strings.
