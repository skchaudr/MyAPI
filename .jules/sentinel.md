## 2026-04-12 - Fix eval command injection in deploy script
**Vulnerability:** The deployment script `deploy_to_brain.sh` contained an unsafe `eval` statement designed to expand the tilde (`~`) character when evaluating user input for a directory path: `eval LOCAL_DIR="$LOCAL_DIR"`. This introduced a critical command injection vulnerability. A malicious actor could provide input like `"; ls -al; echo "` to execute arbitrary commands with the privileges of the script user.
**Learning:** Shell scripts processing user input should avoid the `eval` builtin wherever possible as it evaluates arbitrary code. While `eval` is often tempting for tasks like tilde expansion, safer alternatives exist in bash.
**Prevention:** Rather than utilizing `eval`, use safe bash parameter expansion constructs. In this case, `LOCAL_DIR="${LOCAL_DIR/#\~/$HOME}"` performs a simple pattern substitution, replacing a leading tilde with the user's home directory path without executing the input as a command.

## 2026-06-18 - Fix partial path traversal and information leakage in Imports API
**Vulnerability:** The imports API endpoints (`/codex`, `/claude-code`) passed user-provided path parameters directly to directory scanning functions without validation, creating a partial path traversal vulnerability. Additionally, generic exception handlers were leaking internal error messages directly to the client via `HTTPException`.
**Learning:** Using `os.path.commonpath` effectively ensures that a requested path strictly resides within a base directory, whereas string-based checks like `.startswith()` are vulnerable to partial path traversal (e.g., `/base/dir_malicious` vs `/base/dir`). Failing securely means catching exceptions, logging the stack trace internally (`exc_info=True`), and returning a generic 500 error to the client.
**Prevention:** Always validate user-provided paths by expanding them (`os.path.abspath(os.path.expanduser(path))`) and checking them against a known base directory using `os.path.commonpath`. Avoid exposing raw `str(e)` in error responses to prevent information leakage.
