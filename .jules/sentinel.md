## 2026-04-12 - Fix eval command injection in deploy script
**Vulnerability:** The deployment script `deploy_to_brain.sh` contained an unsafe `eval` statement designed to expand the tilde (`~`) character when evaluating user input for a directory path: `eval LOCAL_DIR="$LOCAL_DIR"`. This introduced a critical command injection vulnerability. A malicious actor could provide input like `"; ls -al; echo "` to execute arbitrary commands with the privileges of the script user.
**Learning:** Shell scripts processing user input should avoid the `eval` builtin wherever possible as it evaluates arbitrary code. While `eval` is often tempting for tasks like tilde expansion, safer alternatives exist in bash.
**Prevention:** Rather than utilizing `eval`, use safe bash parameter expansion constructs. In this case, `LOCAL_DIR="${LOCAL_DIR/#\~/$HOME}"` performs a simple pattern substitution, replacing a leading tilde with the user's home directory path without executing the input as a command.

## 2026-07-15 - Prevent Information Exposure via Error Handling
**Vulnerability:** FastApi router endpoints (`api/routers/enrich.py`, `export.py`, `imports.py`, `query.py`) were returning raw exception details back to the client upon encountering generic exceptions (`raise HTTPException(status_code=500, detail=str(e))`). This could potentially expose sensitive server-side information, file paths, or third-party service specifics to end users.
**Learning:** Exception handling in FastAPI router endpoints should never serialize the raw exception object (`str(e)`) directly to the client response, as it violates the principle of "fail securely" and risks information disclosure.
**Prevention:** Always log the full exception context server-side using `logger.error("...", exc_info=True)` and return a generic, non-descriptive error message (`"An internal server error occurred"`) to the client for generic `Exception` catches. Ensure unit tests assert the generic message instead of the raw exception.
