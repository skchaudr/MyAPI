## 2026-04-12 - Fix eval command injection in deploy script
**Vulnerability:** The deployment script `deploy_to_brain.sh` contained an unsafe `eval` statement designed to expand the tilde (`~`) character when evaluating user input for a directory path: `eval LOCAL_DIR="$LOCAL_DIR"`. This introduced a critical command injection vulnerability. A malicious actor could provide input like `"; ls -al; echo "` to execute arbitrary commands with the privileges of the script user.
**Learning:** Shell scripts processing user input should avoid the `eval` builtin wherever possible as it evaluates arbitrary code. While `eval` is often tempting for tasks like tilde expansion, safer alternatives exist in bash.
**Prevention:** Rather than utilizing `eval`, use safe bash parameter expansion constructs. In this case, `LOCAL_DIR="${LOCAL_DIR/#\~/$HOME}"` performs a simple pattern substitution, replacing a leading tilde with the user's home directory path without executing the input as a command.

## 2026-06-05 - Fix path traversal in import endpoints
**Vulnerability:** The `/codex` and `/claude-code` endpoints in `api/routers/imports.py` accepted a user-provided `root` path parameter and passed it directly to file system operations (via adapter scanning functions) without verifying that the path resolved within the intended base directory. This allowed path traversal (e.g., passing `../../../etc/passwd` to read files outside the intended directories).
**Learning:** Even if the underlying directory scanning logic seems scoped, allowing user-defined `root` directory paths without validation introduces a path traversal risk, especially since path-related inputs often bypass basic text sanitization due to variable filesystem path formats (e.g., using `..` or leading slashes).
**Prevention:** To prevent path traversal in import endpoints or anywhere paths are constructed from user input, expand the given path fully and resolve it to an absolute path (`os.path.abspath(os.path.expanduser(path))`). Then, use `os.path.commonpath([base_dir, target_path]) == base_dir` to explicitly verify that the fully resolved target path originates strictly within the fully resolved base directory.
