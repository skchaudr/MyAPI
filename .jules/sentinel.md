## 2026-04-12 - Fix eval command injection in deploy script
**Vulnerability:** The deployment script `deploy_to_brain.sh` contained an unsafe `eval` statement designed to expand the tilde (`~`) character when evaluating user input for a directory path: `eval LOCAL_DIR="$LOCAL_DIR"`. This introduced a critical command injection vulnerability. A malicious actor could provide input like `"; ls -al; echo "` to execute arbitrary commands with the privileges of the script user.
**Learning:** Shell scripts processing user input should avoid the `eval` builtin wherever possible as it evaluates arbitrary code. While `eval` is often tempting for tasks like tilde expansion, safer alternatives exist in bash.
**Prevention:** Rather than utilizing `eval`, use safe bash parameter expansion constructs. In this case, `LOCAL_DIR="${LOCAL_DIR/#\~/$HOME}"` performs a simple pattern substitution, replacing a leading tilde with the user's home directory path without executing the input as a command.

## 2026-04-12 - Fix path traversal in API import endpoints
**Vulnerability:** The API endpoints for importing codex and claude-code sessions (`/codex` and `/claude-code`) accepted user-supplied directory paths via the `root` parameter in the request payload. These paths were passed directly to `os.path.expanduser()` and then to directory-walking functions like `os.walk()` without any validation. This introduced a path traversal vulnerability, allowing an attacker to read arbitrary `.json` and `.jsonl` files from anywhere on the server filesystem by supplying paths like `../../../../var/log`.
**Learning:** Never pass unfiltered user input directly to file system operations. The `os.path.expanduser()` function simply expands `~` to the user's home directory but does not resolve `../` sequences or check if the resulting path is within an intended scope. Additionally, using string methods like `.startswith()` on paths is dangerous as it allows partial path traversal (e.g., `/app/data` matches `/app/database_backups`).
**Prevention:** Strictly validate user-supplied path parameters by first expanding and resolving them to absolute paths using `os.path.abspath(os.path.expanduser(path))`. Then, verify that the resulting absolute path resides within the intended base directory using `os.path.commonpath([base_dir, target_path]) == base_dir`.
